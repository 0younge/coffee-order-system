# 0014. Outbox 배치를 비동기로 병렬 전송

- 상태: 승인됨
- 날짜: 2026-07-13
- 결정자: 프로젝트 소유자

## 맥락

[ADR 0003](./0003-deliver-order-events-with-transactional-outbox.md)은 한 번에 최대 50건을 선점하고 30초 lease와 5초 응답 timeout을 사용한다. 선점한 이벤트를 순차 전송하면 뒤쪽 이벤트가 호출되기 전에 lease가 만료돼 불필요한 재점유와 중복 전송이 늘어날 수 있다.

## 고려한 대안

- 선점 배치 순차 전송 — 구현이 단순함 / 배치 처리 시간이 lease를 넘을 수 있음
- 고정 스레드 풀에서 동기 전송 — 동시성 상한이 명확함 / 스레드 수와 배치 크기를 함께 조정해야 함
- JDK 비동기 HTTP 전송 — 선점 배치를 즉시 병렬 전송함 / 활성 배치 수를 제한해야 함

## 결정

JDK `HttpClient.sendAsync`로 선점한 이벤트를 병렬 전송한다. 인스턴스마다 활성 배치를 하나만 허용하고 활성 배치가 없을 때 기본 1초마다 최대 50건을 선점한다. 연결 timeout은 2초, 요청 전체 timeout은 5초이며 redirect를 따르지 않는다.

각 HTTP 결과는 별도의 짧은 DB 트랜잭션에서 현재 `claim_token`을 확인한 뒤 반영한다. 인스턴스 로컬의 활성 배치 표시는 과도한 동시 호출을 막는 실행 제어일 뿐이며, 이벤트 소유권과 정합성은 계속 MySQL lease와 fencing이 담당한다.

## 결과

- 선점한 이벤트가 lease 안에 전송을 시작할 가능성을 높이고 순차 처리 지연을 피한다.
- 외부 HTTP 호출은 DB 트랜잭션 밖에 유지된다.
- 인스턴스당 동시 외부 요청이 최대 배치 크기까지 늘 수 있으므로 구현 후 외부 API 용량과 발행 지연을 측정해야 한다.
