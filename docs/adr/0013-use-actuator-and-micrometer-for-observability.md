# 0013. 관측성에 Actuator와 Micrometer 사용

- 상태: 승인됨
- 날짜: 2026-07-13
- 결정자: 프로젝트 소유자

## 맥락

API, DB 경합과 Outbox 상태를 측정할 수 있어야 하지만 현재 빌드에는 공통 지표 수집 수단이 없다. 실제 배포 환경과 외부 모니터링 시스템은 아직 정해지지 않았으므로 exporter나 알림 채널을 먼저 선택하면 현재 범위를 넘는다.

## 고려한 대안

- 로그만 사용 — 추가 의존성이 없음 / 지연 분포와 상태별 지표를 일관되게 집계하기 어려움
- Actuator와 Micrometer — Spring 지표와 사용자 정의 지표를 같은 방식으로 수집함 / 관리 의존성과 설정이 추가됨
- Prometheus exporter까지 도입 — 바로 외부 수집 가능 / 배포·모니터링 시스템 선택을 선행하게 됨

## 결정

`spring-boot-starter-actuator`를 추가하고 Micrometer로 API 지연·오류, DB 풀, DB 경합 오류와 Outbox 상태·재시도를 측정한다. 현재 HTTP 관리 경계에는 상세 정보를 숨긴 health endpoint만 노출한다. 지표 endpoint 공개, 외부 exporter와 알림 연동은 실제 배포 환경을 결정한 뒤 추가한다.

지표 등록과 상태 변화는 테스트의 `MeterRegistry`로 검증하고 요청·주문·이벤트의 연관 정보는 민감정보를 제외한 구조화 로그로 남긴다.

## 결과

- Spring과 애플리케이션 지표를 같은 추상화로 수집하고 자동화 테스트할 수 있다.
- 외부 모니터링 제품 없이도 기능 구현과 성능 기준선 측정에 필요한 계측 지점을 만든다.
- 운영 exporter, 지표 endpoint 접근 통제와 알림 임계값은 후속 배포 결정으로 남는다.
