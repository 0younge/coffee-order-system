# 0019. Spotless를 코드 포맷 완료 게이트로 사용

- 상태: 승인됨
- 날짜: 2026-07-15
- 결정자: 프로젝트 소유자

## 맥락

프로젝트 완료 기준은 테스트뿐 아니라 린트 통과를 요구하지만 현재 Gradle 빌드에는 이를 판정할 도구가 없다. 컴파일과 `check` 통과만으로 린트가 통과했다고 보고할 수 없으며, 사람이 수동으로 포맷을 확인하면 에이전트와 실행 환경마다 결과가 달라질 수 있다.

## 고려한 대안

- 린트 완료 기준 제거 — 새 도구가 필요 없음 / 프로젝트의 기존 완료 기준을 약화함
- Gradle Checkstyle — 다양한 정적 규칙을 검사함 / 규칙 세트 선택과 유지 비용이 늘어남
- Spotless와 Java 포매터 — 결정적인 포맷 검사와 자동 수정을 제공함 / Gradle 플러그인과 포매터 도구가 추가됨

## 결정

Gradle Spotless 플러그인과 Java 포매터를 추가해 Java 소스의 결정적인 포맷을 검사한다. `spotlessCheck`를 `./gradlew check` 완료 게이트에 포함하고, 불일치가 있으면 빌드를 실패시킨다. `spotlessApply`는 개발자가 명시적으로 실행하는 수정 명령으로만 제공하며 검사나 빌드가 소스 파일을 자동 변경하지 않는다.

Spotless와 포매터의 호환 버전은 Java 17·Gradle 9.5.1 기준으로 고정한다. 자동 포맷을 적용한 뒤에는 전체 diff를 검토해 요청 범위 밖 변경이 섞이지 않았는지 확인한다.

## 결과

- `./gradlew check`가 테스트와 코드 포맷을 함께 판정해 린트 완료 기준을 자동화한다.
- 에이전트와 로컬 환경에서 동일한 Java 포맷을 재현할 수 있다.
- Spotless 플러그인과 포매터 버전을 빌드 기준선과 함께 관리해야 한다.
- 정적 버그 분석이나 복잡도 검사는 현재 범위에 추가하지 않으며 필요해지면 별도 도구를 검토한다.
