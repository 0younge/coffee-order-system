# Pull Request

## 절차

1. 현재 브랜치와 base 브랜치를 확인한다. main/master에서 직접 작업한 상태면 먼저 보고한다.
2. `test` 스킬의 완료 기준 통과 후 push → `gh pr create`. (미검증 상태면 draft로 만들고 draft임을 명시)
3. 생성된 PR URL을 사용자에게 전달한다.
4. 사용자 확인 없이 merge하지 않는다.

## 제목·본문

제목은 커밋과 동일한 형식: `<타입>: <한국어 요약>`

본문 템플릿:

```markdown
## 요약
<무엇을, 왜 — 1~3줄>

## 변경 사항
- <핵심 변경만. 파일 나열 금지>

## 테스트
- <실행한 검증 명령과 결과>

## 참고
- <리뷰어가 알아야 할 맥락. 없으면 섹션 삭제>
```
