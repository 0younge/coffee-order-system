# AGENTS.md

AI 코딩 에이전트(Claude Code, Codex, Antigravity 등)가 이 프로젝트에서 따를 행동 지침 정본.
규칙 수정은 이 파일에서만 한다. (`CLAUDE.md`는 `@AGENTS.md` 임포트 포인터)

## 0. 언어

- 모든 응답·설명·질문·코드 주석·커밋 제목은 **한국어**. 변수·함수·클래스명은 영어.
- 에러/로그는 원문 그대로 인용하고 한국어 해석을 덧붙인다. 기술 용어는 원어 병기 가능.

## 1. 코딩 전에 생각하기

- 요구사항이 모호하면 추측하지 말고 먼저 질문한다. 해석이 여러 개면 제시 후 확인받는다.
- 세운 가정은 응답 앞부분에 명시한다. (예: "가정: DB 없음 → 메모리 저장")
- 요청보다 단순한 접근이 보이면 구현 전에 제안한다.
- 이해 못 한 코드는 수정하지 않는다. 읽고 요약한 뒤 수정한다.

## 2. 단순함 우선

- 요청받은 기능만 최소한의 코드로. 미래 대비 기능·추상화·설정 옵션·습관성 방어 코드 금지.
- 표준 라이브러리로 되면 그걸 쓴다.
- 판단 기준: **지워도 요청 기능이 동작하면 지운다.**

## 3. 외과적 수정

- 요청과 직접 관련된 코드만 수정한다. 지나가다 본 포매팅·네이밍은 "김에" 고치지 않는다.
- 내 변경으로 불필요해진 코드는 내가 정리한다. 기존 코드의 스타일·구조 관례를 따른다.

## 4. 목표 중심 실행 · 완료 기준

- 시작 전 완료 기준을 한 문장으로 정의한다.
- 여러 언어·프론트를 쓰므로 "완료"는 범용 기준으로 판정한다:
  **변경 코드의 테스트 통과 + 린트 통과 + 빌드/실행 확인** (상세 절차는 `test` 스킬).
- 코드 작성·변경 = 대응 테스트 동반 작성. 테스트 없는 코드는 미완료 (질적 기준은 `test` 스킬).
- 버그 수정 = 재현 테스트 작성 → 수정 → 통과.

## 경계 (Boundaries)

| 항상 한다 | 먼저 묻는다 | 하지 않는다 |
|---|---|---|
| 수정 후 실행/테스트로 검증 | 새 의존성(라이브러리) 추가 | 요청 없는 force push, `--amend`, rebase |
| 가정·실패·수치를 그대로 보고 | 대규모 리팩터링 | 시크릿(.env, 키, 토큰) 커밋 |
| 커밋 전 diff 전체 검토 | 죽은 코드 삭제 | 통과시키려고 테스트 약화 |

## 스킬 (모든 에이전트 공용)

반복 절차는 `.agents/skills/<이름>/SKILL.md`에 있다. 명시적 요청뿐 아니라 **작업 흐름이
아래 트리거에 도달하면 스스로** 해당 SKILL.md를 먼저 읽고 그대로 따른다.

| 트리거 (요청 또는 상황) | 스킬 |
|---|---|
| 변경을 커밋하게 될 때 | `commit` |
| PR을 만들거나 수정하게 될 때 | `pr` |
| 코드 리뷰·검토를 하게 될 때 | `review` |
| 코드를 변경한 뒤 완료를 판정할 때 (모든 코딩 작업) | `test` |
| 스택·데이터 모델·구조 등 되돌리기 비싼 결정을 하게 될 때 | `adr` |
| AGENTS.md·스킬·실수 노트 등 하네스 파일을 고치게 될 때 | `harness` |

- Codex·Antigravity는 `.agents/skills/`를 직접 인식하고, Claude Code는 `.claude/skills/`의 심링크로 인식한다.

## 실수 노트 (사용자·AI 공동 관리)

같은 실수가 2회 반복되면 AI가 직접 한 줄 추가하고 보고한다. 사용자도 자유롭게 추가·수정한다.
이 섹션은 승인 없이 수정 가능한 유일한 섹션이다. 10개를 넘으면 `harness` 스킬 절차로 정리한다.

- macOS 한글 폴더명은 NFD 유니코드라 경로 직접 입력이 실패할 수 있다 → 글롭/`ls`로 접근한다.

## 프로젝트 정보

- 주제/목표: 다중 인스턴스에서도 포인트·주문 정합성을 보장하는 커피 주문 REST API
- 대상 사용자: 이메일로 인증하고 포인트를 충전해 단일 메뉴를 주문하는 사용자
- 스택: Java 17 / Spring Boot 4.1.0 / MySQL 8.4 LTS / Gradle 9.5.1 / Docker Compose

### 명령어 (플래그까지 정확히)

- 개발 서버: `./gradlew bootRun`
- 로컬 DB: `docker compose up -d --wait`
- 테스트: `./gradlew test`
- 전체 검증: `./gradlew check`

### 고유 규칙

- `docs/adr/`의 제안·승인 결정을 구현 전에 확인하고, 새 구조 결정은 사용자 승인 전 확정하지 않는다.
- 포인트 충전·주문은 사용자 행 비관적 락을 사용하며 외부 HTTP 호출을 DB 트랜잭션에 포함하지 않는다.
- 문서에는 목표 계약과 현재 구현 상태를 구분한다. MySQL Compose와 DB 연결 설정은 구현됐고 Flyway·JWT·업무 기능은 아직 구현 전이다.
